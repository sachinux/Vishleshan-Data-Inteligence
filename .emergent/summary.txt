<analysis>**original_problem_statement:**
The user wants to build a full-stack web app named Data Storyteller Studio. The primary goal is to allow users to upload data files (CSV, Excel, Google Sheets, PDF), analyze them using natural language queries in a chat interface, and generate interactive charts and insights. These insights can be compiled into Story Tiles and arranged on a Storyboard, which can then be exported. The application should have a modern React frontend and a FastAPI backend, using an LLM for language understanding and summarization, while performing actual data analysis with code (e.g., pandas).

The user's vision has evolved to focus heavily on making the generated stories (Data Actions) into actionable, decision-making tools with features like KPI dashboards, action item checklists, and stakeholder-specific views.

**PRODUCT REQUIREMENTS:**
-   **File Handling:** Accept CSV, Excel, Google Sheets (via URL), and PDF uploads.
-   **Data Profiling:** Automatically analyze and display data schema, types, nulls, etc.
-   **Chat-based Analysis:** A chat UI where users can ask questions about their data.
-   **Visualization:** Auto-generate interactive charts (bar, line, scatter) from analysis results.
-   **Data Grid:** An Excel-like grid view of the data where users can select/multi-select rows to generate a story or comparison.
-   **Actionable Storytelling (Data Actions):**
    -   **Pinned Insights:** Create tiles from key insights found during chat analysis.
    -   **Data Actions (formerly Storyboard):** Generate a presentation from selected insights that includes an executive summary, a KPI dashboard, a prioritized and trackable action item checklist, and tailored views for different stakeholders (e.g., Executive, Manager).
-   **Export:** Export the final Data Actions to PDF or PPTX (JSON is acceptable initially).
-   **Architecture:** React frontend, FastAPI backend, MongoDB database. LLM for NLP tasks only.

**User's preferred language**: English

**what currently exists?**
A full-stack application with a React frontend and FastAPI backend. The UI is a three-column layout: a left sidebar for navigation and data management, a central area for chat/analysis (with a toggleable data grid split-view), and a tab-based right sidebar for viewing Pinned Insights and a Quick Actions dashboard.

Key implemented features include file uploading, data profiling, chat-based analysis, an Excel-like data grid with row selection, chat settings, and the generation of highly interactive Data Actions from pinned insights. These Data Actions include an executive summary, KPIs, a trackable action item checklist, and stakeholder views. The UI also supports deleting insights and Data Actions.
